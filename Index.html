Overview

This app has two main workflows. The first workflow is the standalone web app on this page, which creates a new Drive workspace with basic Docs, Sheets, and Slides. The second workflow is the spreadsheet-bound template generator, which uses a Google Sheet plus the sidebar to create many personalized Docs, Slides, Sheets, or PDFs from template IDs.

For the most complete experience, use this project as a Google Sheet-bound Apps Script project and run the demo setup first.

Standalone web app workflow

  1. Enter a workspace name and optional description.
  2. Choose how many Docs, Sheets, and Slides to create.
  3. Click Preview JSON if you want to inspect the generated configuration.
  4. Turn on Use advanced JSON if you want custom file names, document bodies, sheet values, or presentation titles.
  5. Click Generate workspace.
  6. Open the returned folder and file links from the Results panel.
Advanced JSON structure

The advanced JSON may include workspaceName, description, documents, spreadsheets, and presentations. The app limits very large runs so the Apps Script execution remains manageable.

Spreadsheet-bound generator workflow

  1. Push or copy the root project files into a Google Apps Script project bound to a Google Sheet.
  2. Reload the spreadsheet.
  3. Open Productivity Suite > Setup Complete Demo Workspace.
  4. The setup creates demo templates, a Drive folder, and the required sheets.
  5. In R-DOC-GEN, edit readable labels in row 2; row 1 placeholders update automatically.
  6. In C-DOC-GEN, edit readable labels in column B; column A placeholders update automatically.
  7. Select valid rows in R-DOC-GEN or valid item columns in C-DOC-GEN.
  8. Open Productivity Suite > Run Health Check.
  9. Open Productivity Suite > Open Generator Sidebar.
  10. Choose output options and click Generate.

R-DOC-GEN row mode

Use row mode when each selected row represents one output file.

AreaPurpose
Column ATemplate file ID. Use a Google Docs, Slides, or Sheets template ID.
Column BDestination Drive folder ID.
Column CRecipient email. Optional.
Column DOutput name.
Row 2, columns E+Readable field labels such as Name, Score, or Activity.
Row 1, columns E+Formula-generated placeholders such as ={{"{{"}}&E2&{{"}}"}}, displayed as {{Name}}.
Rows 3+Actual generation data. Select rows starting from row 3.

Edit the readable labels in row 2; the placeholder headers in row 1 update automatically.

C-DOC-GEN column mode

Use column mode when each selected item column represents one output file.

AreaPurpose
Row 1, columns C+Template file ID per item column.
Row 2, columns C+Destination Drive folder ID.
Row 3, columns C+Recipient email. Optional.
Row 4, columns C+Output name.
Column B, rows 5+Readable field labels such as Title, Class Code, or Course Number.
Column A, rows 5+Formula-generated placeholders such as ={{"{{"}}&B5&{{"}}"}}, displayed as {{Title}}.
Columns C+Actual item columns. Select item columns starting from column C. Column B is only a helper.

Edit the readable labels in column B; the placeholder labels in column A update automatically.

Placeholders and formatting

Placeholders use the field name inside double braces. Formatting modifiers are placed before the final closing braces.

PlaceholderEffectExample output
{{Name}}Normal replacementjuan dela cruz
{{Name}u}}Uppercase / all capsJUAN DELA CRUZ
{{Name}b}}Boldjuan dela cruz
{{Name}ub}}Uppercase and boldJUAN DELA CRUZ
{{Name}1}}Title caseJuan Dela Cruz
{{Name}1b}}Title case and boldJuan Dela Cruz
Docs and Slides support bold formatting as styled text. Emails use HTML bold. Sheets replace text values, but do not apply rich-text bold formatting inside cells.
Image placeholders

If a replacement value is an image URL or an accessible Drive image file ID, the generator can insert it as an image in Docs or Slides where supported.

Output modes

Separate output

Each selected row or item column creates its own generated file. This is the safest default for mixed Docs, Slides, and Sheets templates.

Combined Docs or Slides

Use combined Doc/Slide mode only when all selected templates are compatible. Do not mix Docs and Slides in one combined Doc/Slide run.

Combined Sheets

Use combined Sheet mode to copy selected Sheet templates into one spreadsheet. Sheet names are automatically sanitized and made unique.

PDF output

PDF conversion applies to generated Docs and Slides. Sheet templates remain Google Sheets unless separate PDF export support is added later.

Email and logs

The EMAIL sheet stores the optional email subject and body templates. If the sidebar option to send email is enabled and a selected item has a recipient email, the app can email the generated file.

The Log sheet records timestamp, generator type, file name, file URL, folder URL, recipient, email status, generation status, and details.

Health check, cancel, and reset

  • Run Health Check validates required sheets, active selection, placeholder setup, template IDs, folder IDs, and external request access.
  • Cancel Current Generation stops the current queued generation state.
  • Reset Generator State clears stored generation state and pending triggers. Use this if a test run gets stuck.

Testing checklist

  • Confirm the Productivity Suite menu appears after reloading the Google Sheet.
  • Run Setup Complete Demo Workspace.
  • Confirm the demo Drive folder and sample templates are created.
  • Confirm readable labels create placeholders automatically in R-DOC-GEN row 1 and C-DOC-GEN column A.
  • Select R-DOC-GEN rows 3:5 and run Health Check.
  • Generate separate files from row mode.
  • Select C-DOC-GEN item columns C:E and run Health Check.
  • Generate separate files from column mode.
  • Test invalid template IDs and folder IDs to confirm preflight validation stops the run.
  • Test PDF output with Doc or Slide templates.
  • Test email sending using your own email address first.

Troubleshooting

The menu does not appear

Reload the Google Sheet. Confirm the Apps Script project is bound to the Sheet and that root files were pushed correctly.

Health check says a template ID is invalid

Check that the value is only the file ID, not the full URL. Also confirm your account has access to the template file.

Health check says a folder ID is invalid

Check that the destination folder exists and your account can write files into it.

Column mode does not detect column B

Column B is intentionally only a readable field-name helper. Select item columns starting from column C.

Combined output fails

Use combined modes only with compatible template types. Do not combine Docs and Slides in one combined Doc/Slide run.

Email fails but file generation succeeds

Confirm the recipient email is valid and that the script has Mail scope authorization. Start by emailing your own address.

Generation seems stuck

Use Productivity Suite > Reset Generator State, then run Health Check before starting again.
